# Dokumentace: `i18n.js`

Modul slouží k **internacionalizaci** (lokalizaci) aplikace do více jazyků. Podporuje momentálně češtinu (`cs`) a angličtinu (`en`). Není závislý na externích velkých knihovnách třetích stran (jako i18next), ale načítá lokalizační slovníky přímo ze statických JSON souborů (např. `/locales/cs.json`).

## Základní mechanismy a exporty

- **`t(key, vars = {})`**: Hlavní funkce tohoto modulu. Vrátí přeložený řetězec na základě zadaného klíče `key`. Podporuje také dosazování proměnných (placeholderů).
  - *Použití:* `t('pozdrav', { jmeno: 'Pavel' })` nahradí tag `{jmeno}` v překladu hodnotou 'Pavel'.
  - Pokud klíč není nalezen, funkce vrací samotný klíč, aby se zamezilo pádu a UI bylo hned nápovědné k doplnění.
- **`applyTranslations()`**: Najde v dokumentu všechny HTML prvky obsahující atribut `data-i18n` a pokusí se je přeložit. Volitelně zvládá překládat i vlastnosti daných prvků, jako například `placeholder`, pokud je u nich definován sub-atribut `data-i18n-attr`. Rovněž změní nadpis stránky (`document.title`) a atribut `lang` značky `<html>`.
- **`setLanguage(lang)`**: Asynchronně změní aktuální jazyk. Stáhne (nebo převezme z mezipaměti objektu `cache`) nový JSON překladů, zapíše výběr do `localStorage` a zavolá `applyTranslations`. Nakonec vyvolá vlastní globální událost `langchange` přístupnou pro ostatní moduly a view komponenty.
- **`toggleLanguage()`**: Jednoduše "prohazuje" mezi podporovanými jazyky (`cs` a `en`).
- **`initI18n()`**: Zpřístupní modul pro zbytek kódu. Volá se jako první inicializační operace v `app.js`.
